# 城市洪涝灾害多模态社交媒体数据分析系统开发白皮书

> **系统定位**：构建一个端到端的自动化感知与分析流水线，利用前沿大模型技术从无结构社交媒体流中抽提洪涝灾害实体要素，并最终实现与地理空间及遥感监测数据的跨源融合。
> **核心架构**：基于“采集-清洗-多模态抽取-时空融合”的四阶段解耦架构。系统以**领域知识驱动（Domain Knowledge Driven）**为底层逻辑，取代传统庞杂的规则系统，确保系统在复杂气象地质场景下的泛化能力与学术延展性。

## 模块一：智能数据采集模块 (Intelligent Ingestion)

本模块摒弃传统基于固定DOM解析的脆弱爬虫，转向基于大语言模型友好的语义抓取框架。

* **1.1 核心驱动：`Crawl4AI` 智能框架**
* **动态渲染与语义提取**：利用 `Crawl4AI` 原生支持的异步浏览器管理，处理瀑布流加载。通过框架内部的 LLM 提炼能力，直接将网页源码（HTML）转化为纯净的 Markdown 格式，剥离广告、侧边栏和无关的导航元素。
* **自适应反爬与代理路由**：内置智能重试与退避算法，支持动态代理池，确保在高并发获取突发灾害数据时的网络稳健性。


* **1.2 动态多维策略控制**
* **事件驱动的时间窗**：不再依赖固定的定时任务。系统接入气象预警接口（如暴雨红色预警），在特定区域触发预警时，动态缩短该区域的采集滑动窗口，实现“战时”高频定向拉取。
* **多层级信源矩阵**：根据发布者权重（政务号、新闻媒体、实地打卡个人用户）分配不同的信任度先验分数，为后续的置信度评估提供基准。


## 模块二：数据清洗与高层过滤模块 (Cleaning & Filtering)

社交数据的信噪比极低，本模块旨在以分层过滤策略，在保证召回率的同时大幅降低下游大模型的推理成本。

* **2.1 文本与多模态去重**
* **局部敏感哈希 (LSH)**：采用 MinHash 或 SimHash 算法计算文本指纹，快速识别跨平台的同质化文本（如复制粘贴的灾情通报）。
* **感知哈希 (pHash)**：对于配图数据，提取图像哈希值以过滤重复搬运的灾害现场照片。


* **2.2 大模型灾害相关性与意图分类**
* **Zero/Few-Shot 提示词工程**：利用轻量级 LLM（如 Qwen 1.5/Llama 3）替代传统的词典/正则规则。通过精心设计的 Prompt，让模型理解文本的真实意图。
* **多分类输出**：精确区分“事实性灾情报道”、“灾害求救”、“泛泛的情绪宣泄”与“无关比喻（如‘被作业淹没’）”。针对具有导流、造谣特征的高风险模式，输出风险等级标签及判定依据。



## 模块三：多模态信息提取模块 (Multimodal IE)

本模块是系统的核心智能引擎，负责将非结构化的图文片段转化为后续融合分析所需的结构化事实依据。

* **3.1 文本要素的强制结构化 (Text-to-Schema)**
* 结合 `Pydantic` 或 `Instructor` 等结构化输出库，强制 LLM 输出符合预定义 Schema 的 JSON 数据。
* **抽取要素**：包含灾害类型（积水、倒灌、道路阻断）、相对/绝对时间归一化、以及核心的**受灾位置描述**和**灾情程度量化（水深估算）**。


* **3.2 图像视觉验证与少样本学习 (Visual IE)**
* **面向新场景的二分类**：对于社交媒体附带的灾害图像，系统重点利用少样本学习（Few-shot learning）进行洪涝/非洪涝的二分类识别。
* **算法设计要求**：该视觉模块侧重于对突发新场景的快速适应。在模型设计上，不要求保留原始基础类的分割或识别能力（即不涉及基础知识的遗忘问题），而是直接利用少样本快速在新数据集上建立二分类边界。实验对比阶段，直接选取 5-6 个经典的少样本分类模型进行性能基准测试，避开与庞大繁冗的基础模型（Foundation Models）进行对比，确保工程落地的轻量与高效。



## 模块四：融合分析模块 (Fusion & Spatial Intelligence)

将提取出的离散社交媒体事件锚定到真实的物理空间，并作为多模态预训练模型的数据补充或交叉验证源。

* **4.1 空间拓扑与动态地理编码**
* **高德/百度 API 接入**：将文本抽取的候选地点（POI）转化为精确的 WGS84/GCJ02 经纬度坐标。
* **模糊位置落点**：针对“某路与某路交叉口”、“某小区地下车库”等相对位置，结合地理拓扑关系进行概率落点估计。
